数据库系统概念第五章——高级SQL笔记

1. JDBC

​ 针对Java连接数据库服务器的应用接口。是当下流行的数据库操纵技术。

2. ODBC

​ ODBC是由微软提供的标准API来访问数据库，如果数据库支持ODBC标准，那么用户就可以使用ODBC的API来访问它。常见的主流数据库（Mysql，SQLserver等）都支持ODBC标准。

​ 关于ODBC的资料非常少，其中比较详细的资料有《ODBC API开发教程》（作者：闻怡洋），但这本书是2002年的时候写的。现在主流的开发已经很少用ODBC来操作数据库了，在书上也是一笔带过。

3. 嵌入式数据库

​ 嵌入式数据库指的是：将SQL代码写入高级语言中（比如C、C++、JAVA、Fortran中），这里的高级语言也被称为“宿主语言”，程序在编译前由一个特殊的预处理器进行处理，然后再经过宿主语言的编译器进行编译。

3.1 基本语法

​ 嵌入式SQL的基本语法如下：

EXEC SQL ;

​ 对于不同的宿主语言，语法各不相同。比如在COBOL中，分号被替换为END-EXEC。而JAVA中，采用# SQL {...};的书写形式。

​

3.2 连接数据库

EXEC-SQL connect to server user user-name using password;

3.3 变量共享

​ 嵌入式SQL中可以使用宿主语言的变量，但是前面要加上冒号:来区别于SQL变量。

​ 这种变量必须声明在如下的区段中，如：

EXEC SQL BEGIN DECLARE SECTION;
int credit_amount;
EXEC SQL END DECLARE SECTION;

3.4 查询和修改

​ 要表示关系查询，我们使用一个叫“声明游标”（declare cursor）的东西，它其实和我们绝大多数高级语言操纵数据库的方式是类似的。

EXEC SQL
    declare c cursor for
    select ID,name
    from student
    where tot_cred > :credit_amount
END_EXEC

​ 之后，我们就可以通过对c的迭代循环去获取查询结果。

EXEC SQL open c;
EXEC SQL fetch c into :si, :sn END_EXEC
EXEC SQL close c;

​ 上面的si和ni分别对应着ID和name属性。
​ 一条单一的fetch只能得到一个元组，因此我们可以用一个while循环拿到所有的结果。

​ 因为修改不返回结果，所以相对语法较为简单：

EXEC SQL <合法的update SQL语句>;

3.5 总结

​ 其它的操作（如事物、权限、锁等）就不一一列举了，总之，嵌入式SQL为我们提供了访问数据库的一种方式。

​ 当然，现在大部分用到数据库的开发项目几乎很少用到嵌入式SQL（除非是非常非常简单的项目），网上关于嵌入式SQL的学习资料也很少。由此可以看出，这项技术正在慢慢被淘汰。

4. SQL中的触发器

​ 触发器是一条语句，当对数据库进行修改时，它会自动被系统执行。

4.1 触发器的定义

​ 触发器定义前需要确定三个问题：

  1. 什么条件触发？（insert、delete、update）

  2. 什么时候触发？（修改前/修改后）

  3. 触发频率（针对每一行for each row还是每一次操作for each statement）

注意：触发器定义在表上且附着于表（不可以是临时表）上！

效率上的考虑：尽量少用触发器，如果要使用，要保证触发器体内的语句执行效率尽可能高。

​ 基本格式：

create trigger <trigger_name>
[before|after]
[delete|update|insert]
on <table_name>
/* referencing new row as nrow 或 referencing old row as orow*/
for earch row /*  for each statment在mysql中不支持 */
begin
    ...
    ...
end;

​ 其中，before和after分别表示begin...end内的语句在修改前还是修改后执行。

​ delete、update、insert表示触发条件。

​ delete中只能引用old，表示已经被删除的元组。

​ update中的new表示update后的元组，old表示update前的元组。

​ insert中只能引用new，表示将要插入的元组。

​ referencing new row as <新row名> 的语法只是sql标准，在每个不同的sql产品中定义各不相同。

​ 如果要撤销所有操作，可以在begin...end中添加rollback关键词。

​

4.2 触发器示例

​ 对于给定如下的关系模式bank：

​ 请定义一个触发器，在删除一个账户account时，检查每一个拥有这个账户的户主是否还有剩余的账户，如果没有，那么将它从depositor中删除。

/* 写法一 */
create trigger trigger1
after
delete
on account
referencing old row as orow
for each row
begin
    declare cnt integer default 0;
    select count(*) into cnt
    from depositor
    where depositor.account_number=orow.account_number;
    if cnt = 0 then
        delete from depositor
        where depositor.account_number = orow.account_number;
    end if
end
/* 写法二 */
create trigger trigger1
after
delete
on account
referencing old row as orow
for each row
begin
    delete from depositor
    where orow.customer_name not in (
        select customer_name
        from depositor
        where account_number <> orow.account_number
    )
end

4.3 触发器的查询

show triggers;

​ 但无法查询指定trigger的信息。

4.4 删除触发器

drop trigger [IF EXISTS] [schema_name.] trigger_name;

4.5 for each statement和for each row

​ 在Mysql中，触发器的定义必须使用for each row，它的意思是，对于每一个被影响的行，都要去执行触发器体内的语句。

​ 如果我们只想执行单一动作，对所有被影响的行做一次操作，而不是对每个被影响的行执行一个动作的话，那么就用for each statment。

​ 中间的临时表用referencing old table as <oldtable-name>或referencing new table as <newtable-name>，它对于涉及多行update的操作非常有效。
