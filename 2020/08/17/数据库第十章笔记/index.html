数据库系统概念第十章——存储和文件结构笔记

1. 文件组织

1.1 块（block）

​ 一个数据库被映射为多个文件，每个文件分成定长的存储单元，称为块（block）。

​ 块是存储、传输的基本单元。大多数数据库默认使用4~8KB的块大小。

1.2 定长记录

​ 一个块可能包含多条记录，记录可以被认为是一个元组。记录分为定长记录和变长记录，我们先讨论定长记录的存储。

​ 考虑以下的定义（假设给每个varchar类型分配最大字节数）：

type instructor = record
    ID varchar(5);
    name varchar(20);
    dept_name varchar(20);
    salary numeric(8,2);
end

​ 显然，一条instructor类型的记录占用53个字节。因此，除非块的大小是53的倍数，否则一些记录会跨过块的边界（记录的一部分在一个块中，另一部分在另一块中）。为了避免这个问题，我们让每一块都必须完整保存每一条记录，块中余下的字节放弃使用。

​ 关于维护有两种办法：1. 循环移动记录进行插入删除 2.维护空闲链表（Free List）

1.3 变长记录

​ 变长记录分为两个部分：定长属性（日期、数字、定长字符串）和变长属性（varchar）。

​ 对于定长属性，我们只需要分配定长属性所占用的固定空间即可。

​ 对于变长属性，我们要在记录的开头记录一对值(偏移量,长度)。其中，偏移量表示在记录中，属性数据的开始位置；长度表示变长属性占用的字节数。

2. 文件中记录的组织

常见的组织方法有如下几种：

  1. 堆文件组织
  2. 顺序文件组织
  3. 多表聚簇文件组织
  4. 散列文件组织（索引章讲）
  5. B+树文件组织（索引章讲）

2.1 堆文件组织

​ 记录可以被放在文件的任何空闲位置，同时被放置后就不可被移动。

​ 因此要能够有一种手段去维护文件内所有的空闲空间。

空闲空间图（Free Space Map）

​ 它用来记录本文件有多少个空闲块，每个块中剩余的空闲空间有多少。

​ 顺序结构能够进行高效的查询，它按照搜索码来进行搜索。

​ 顺序不一定在物理空间上连续，因为它是用指针连接起来的，但要尽可能保证物理空间上接近（减少跨块访问）。

3. 表的划分

​ 将一张大的表分割成若干个小表，比如按照年份来划分：将表transaction分为transaction_2019和transaction_2020。

4. 数据字典

​ 数据字典用于存储元数据（关于数据的数据），比如有：关系名、属性列、索引、完整性约束、视图定义、触发器定义、数据库用户等等。

5. 数据库缓冲区

​ 缓冲区是指内存中存储磁盘文件块的拷贝。

​ 当需要磁盘中的某一块时，调用缓冲区管理器（Buffer Manager）对缓冲区进行管理，将需要的块调入缓冲区（这中间也包含替换策略）。

​ 这里牵涉到3种技术：

  1. 缓冲区替换策略：略。
  2. Pinned Block：当一个块正在更新时，大多数恢复系统禁止块写回磁盘，此时该块被认为是”Pinned Block”。
  3. 强制写出：某些情况下，我们需要将缓冲区内的块写出到磁盘。（因为崩溃时，缓冲区的数据会丢失而磁盘不会）

​ Pinned Block和强制写出技术将会在恢复系统章节中阐述。
